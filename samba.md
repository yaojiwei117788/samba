在 Armbian 上配置 Samba 以实现匿名登录的步骤如下：

---

### **1. 安装 Samba**
首先确保系统已更新：

sudo apt update && sudo apt upgrade -y


安装 Samba 包：

sudo apt install samba -y


---

### **2. 创建共享目录**
选择一个目录作为共享目录（例如 `/home/share`），并设置适当的权限：

sudo mkdir -p /home/share
sudo chmod 755 /home/share
sudo chown root:root /home/share


如果希望匿名用户可写入，可放宽权限：

sudo chmod 777 /home/share

⚠️ **注意**：`chmod 777` 会开放所有用户的完全访问权限，需确保目录内不存放敏感数据。

---

### **3. 配置 Samba**
编辑 Samba 的配置文件 `/etc/smb.conf`：

sudo nano /etc/smb.conf


#### **全局设置（Global Section）**  
在 `[global]` 区块中添加以下内容以启用匿名访问：

[global]
   workgroup = WORKGROUP
   server string = Armbian Samba Server %v
   security = user         # 默认为用户验证，但后续共享项会覆盖此设置
   map to guest = bad user # 允许将无效用户名转换为guest账户

   # 以下选项可根据需要调整
   log file = /var/log/samba/log.%m
   max log size = 1000


#### **添加匿名共享配置**
在文件末尾添加一个共享目录的配置项（例如 `[shared]`）：

[shared]
   comment = Anonymous Share
   path = /home/share          # 共享的物理路径
   browseable = yes            # 允许浏览
   writable = yes              # 允许写入
   guest ok = yes              # 启用匿名访问
   read only = no              # 取消只读（如果需要可写）
   create mask = 0755          # 新建文件权限
   directory mask = 0755       # 新建目录权限


---

### **4. 重启 Samba 服务**

sudo systemctl restart smbd
sudo systemctl enable smbd    # 设置开机自启


---

### **5. 防火墙设置（如启用）**
如果系统启用了防火墙，开放 SMB 端口：

# 使用 ufw 的话：
sudo ufw allow smb       # 或直接指定端口 445
sudo ufw reload

# 或者 iptables 方式：
sudo iptables -A INPUT -p tcp --dport 137:139 -j ACCEPT
sudo iptables -A INPUT -p udp --dport 137:139 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 445 -j ACCEPT


---

### **6. 测试共享**
#### **从 Windows 访问**：
在地址栏输入 `\\<Armbian_IP>`, 即可看到共享目录 `[shared]`，无需用户名密码即可访问。

#### **从 Linux 终端测试**：

# 列出共享列表
smbclient -L //<Armbian_IP> -N

# 连接共享（匿名）
smbclient '//<Armbian_IP>/shared' -N


---

### **注意事项**
1. **安全性风险**：匿名访问会降低安全性，建议仅在私有局域网使用，并确保共享目录不包含敏感数据。
2. **权限控制**：若需限制写入权限，可将 `writable = yes` 改为 `no`，或调整 `create mask` 和 `directory mask` 的值（如 `0644`, `0755`）。

---

### **常见问题排查**
- 若无法连接，请检查：
  - Samba 服务状态：`systemctl status smbd`
  - 防火墙是否阻止端口。
  - 共享目录权限是否允许读写（例如 `chmod 777` 或调整用户组）。

希望以上步骤能帮助你配置匿名访问的 Samba！如果有任何错误或疑问，请随时反馈。
